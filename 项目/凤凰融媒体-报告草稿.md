# 关于建设「凤凰卫视融媒体内容中台」的报告

---

## 一、现状问题

目前我台在各社媒平台（国内及海外）拥有大量官方账号，内容发布的工作流程如下：

> **编辑制作视频 → 手工分发给各平台运营 → 运营分别手动发布 → 手工收集数据**

这个流程存在三个突出问题：

**1. 效率低** — 一个视频要在多个平台发布，需要编辑和运营反复沟通、重复操作，人力成本高。

**2. 分散乱** — 账号分散在各个部门和关联机构，谁在管、发没发、发在哪，底数不清。

**3. 数据盲区** — 视频发出去之后，在各平台的传播效果（播放量、点赞、评论等）难以全面及时掌握。

---

## 二、总体方案

### 核心逻辑

> **编辑一次制作，领导线上审签，系统自动分发，数据统一回传。**

### 工作流程

```
  ┌──────────────┐
  │  编辑上传视频  │
  │  填写多平台文案 │
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │  领导审看签发  │  ← 新增：线上审核，通过或驳回
  │  一键批准发稿  │
  └──────┬───────┘
         │ 审核通过
         ▼
  ┌──────────────┐
  │  系统自动分发  │
  │  抖音 │ 快手    │
  │  B站 │ 微博    │
  │  视频号 │ …   │
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │  数据自动回传  │
  │  播放量 │ 点赞  │
  │  评论 │ 分享   │
  │  统一看板展示  │
  └──────────────┘
```

### 平台做三件事

| 模块 | 做什么 | 解决什么问题 |
|---|---|---|
| **统一审片发稿** | 编辑上传→领导审核→一键签发→自动分发 | 人工传递慢、流程不可控 |
| **统一账号管理** | 所有社媒账号集中管理、权限清晰 | 账号散落、底数不清 |
| **统一数据看板** | 各平台数据自动回传、汇总展示 | 数据盲区、决策无依据 |

---

## 三、覆盖平台

结合凤凰卫视的媒体定位和全球传播需求，平台覆盖分国内、海外两大板块：

### 国内平台

| 平台 | 技术条件 |
|---|---|
| **抖音** | 开放平台API支持自动发布和数据查询 ✅ |
| **快手** | 开放平台API支持自动发布和数据查询 ✅ |
| **B站** | 开放平台API支持视频管理和数据开放 ✅ |
| **微博** | 发帖API已逐步下线，需确认可行方案 ⚠️ |
| **微信视频号** | 通过微信开放平台可实现发布和数据回传，但平台仍在快速迭代，接口存在变动可能 ⚠️ |

### 海外平台

| 平台 | 技术条件 |
|---|---|
| **YouTube** | 开放API（YouTube Data API）支持上传发布和数据查询，技术上最成熟 ✅ |
| **Facebook** | Facebook Graph API支持视频发布和数据回传 ✅ |
| **Instagram** | Instagram Basic Display / Graph API支持内容发布 ✅ |
| **X (Twitter)** | X API支持媒体发布和数据分析 ✅ |
| **海外版TikTok** | TikTok Business API支持发布和数据查询 ✅ |

凤凰卫视全球记者站遍布五大洲，海外内容生产能力充足。而海外社媒平台的开放程度普遍高于国内平台，技术实现上反而更快。两者同步建设，打通国内国际传播双通道。

---

## 四、预期效果

| 对比项 | 现状 | 建成后 |
|---|---|---|
| **内容流转** | 编辑逐一分发，沟通成本高 | 线上审签，系统自动推送 |
| **多平台发布** | 运营分别手动操作 | 一次编辑，国内外平台全自动分发 |
| **数据追踪** | 人工收集，时滞长、不全面 | 自动回传看板，实时可查 |
| **账号管理** | 分散各部门，底数不清 | 统一管理，权限分明 |
| **传播覆盖** | 国内为主，海外运营力量不足 | 国内国际双通道，一键直达全球受众 |

### 具体收益

- **节省人力** — 编辑一次操作，多平台自动发布，减少运营重复工作
- **提效增速** — 从"小时级"分发缩短到"分钟级"
- **数据驱动** — 各平台传播数据统一展示，内容效果一目了然
- **风险可控** — 账号集中、权限清晰，避免管理混乱导致的合规风险

---

## 五、实施建议

分两步走，小步快跑、验证后再铺开：

**第一步：试点（1-2个月）**
- 投入：1-2名开发人员，1名外联人员
- 目标：选一个平台（建议抖音）做试点，打通"上传→审核→分发→数据回传"全流程
- 产出：验证技术可行性，积累经验

**第二步：全面铺开（3-6个月）**
- 投入：2-3名开发人员，1名外联人员
- 目标：扩展到快手、B站、微博、视频号、微信视频号等国内主要平台，同步接入YouTube、Facebook、Instagram等海外平台
- 产出：完善数据看板，实现国内国际双通道的统一内容发布管理

---

## 六、平台命名建议

建议命名为 **「凤凰卫视融媒体内容中台」**，呼应融媒体战略方向，对内对外口径统一。
